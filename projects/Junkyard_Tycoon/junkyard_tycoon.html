Junkyard Tycoon Project

Start Date: October 25th, 2025 – Finish Date: In Development



Game Idea

The concept for this game is that the player lives in a junkyard where they can explore and collect various junk items. Junk can be sold or repurposed to earn money or craft new items.
As the player’s business grows and becomes more successful, they can reinvest their profits to expand and improve the junkyard. I might even make a crafting feature of some kind for repairing parts of the junkyard, for example a forklift.


Player Mechanics

  • Walking – The player can freely move in all directions from a top-down, bird’s-eye perspective using the WASD keys or arrow keys.
  • Inventory System – Players can store multiple small items in their inventory or carry one medium-sized item in their hands.
  • Large Item Handling – A forklift is planned for transporting one large item at a time.

Core Mechanics

  • Sales – Players can quick-sell items for a fixed price.
  • Auctions – The auction system allows players to potentially sell items for a higher price, but it takes longer and carries the risk of selling for less.
  • [Possible Feature]Scrap for Parts – The crafting system lets players break down items for materials and use them to unlock new parts of the game, such as building a forklift or dolly.

People Involved in Project

This is a solo project that I’m building entirely from scratch. I’m not an artist, so I’m learning as I go and experimenting with simple design techniques.


Character Customization

One of my main goals with this project is to challenge myself and explore new systems, which is why I decided to make the player fully customizable. This includes options for skin tone, clothing, hats, hairstyles, and facial features.

Since I’m not an artist and creating a full player sprite from scratch is already challenging, I approached it in a modular way. I first created a blank white base sprite for the player. Then, in Unity, I tint each individual piece of the character. For layers like the shirt, pants, hair, and face, I created separate objects so each part can be tinted and customized independently.

I’m not entirely sure how most games handle character customization, but this approach made the most sense for me and goals for this project.

Money Management System

The Money Management System handles the player’s in-game currency using three main functions: GiveMoney, RemoveMoney, and SetMoney. The first two functions take a value as a parameter and adjust the player’s balance accordingly. The SetMoney function is used only at the start of a new save to give the player some starting cash.

These functions will integrate with the game’s buying and selling mechanics to update the player’s balance automatically when needed.

Player money will also be saved to preserve progress between sessions. Since the game is single-player, I don’t plan to include any file protection. it’s up to the user if they want to modify their save data. My main goal is to make sure the data is properly stored, retrieved, and updated as the player progresses.


Inventory System

The inventory system tracks and displays the items the player is carrying, which must also persist between sessions like the player’s money.

The player will have a limited number of inventory slots, encouraging strategic item management and leaving room for future upgrades.

Since the game is 2D, there’s no need to lock the player’s mouse. Because of this I plan to let players rearrange items in their hotbar by dragging and dropping them at the bottom of the screen.


Junk Spawn Locations & Generation

The junkyard will have fixed spawn points for different quality levels of junk, high-end, mid-range, and low-end plots, rarer items spawn in harder to reach places. I think all plots will have a chance to spawn both good and bad items, but high-end plots will have better odds than low-end plots. Once trash spawns, it won’t disappear on its own; the player must deal with it manually. Each item has a different weight some can be carried in the player’s backpack, others require a dolly or even a vehicle to move. Trash has a chance to spawn each night using a random number generator for each plot.


GameManager Object:

The GameManager stores item IDs and sprites for each item, tracks all spawn points, and randomly generates what items appear, where they appear, and their rarity. Items only spawn if a plot is empty; if it’s occupied, the player misses that night’s spawn. This system encourages players to clean up junk regularly while rewarding them when high-value items appear.


SpawnPoint:

Each SpawnPoint includes a script that triggers when the player interacts with it. The script checks whether the player can pick up the item or not. if not, this script will display a message explaining why. If the player can pick it up, the script notifies the GameManager to mark that plot as empty and add the item to the player’s inventory.


Locked Regions

I plan on having loced areas of the junkyard which the player can explore later in the game. These areas will probably be blocked by things like fallen trees or huge piles of trash which the player has to pay to remove, these areas will have higher valued loot and maybe new systems to explore.
